[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture of how this agent operates. When a patient submits a request, it first enters our Security Checkpoint. This node handles PII scrubbing, prompt injection prevention, and medical abuse checks. If cleared, the query proceeds to our central Orchestrator Agent. The orchestrator coordinates specialized tasks by calling the scheduler_agent and interaction_agent as functional tools. The interaction_agent queries our Model Context Protocol server, which acts as a clinical database. If the MCP check returns a severe warning, our Workflow pauses at the Human-in-the-Loop gate, yielding a request for user confirmation. Once approved, the flow resumes and routes to the explainer_agent, which translates the technical findings into simple language before delivering the final response.
